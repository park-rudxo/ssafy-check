# SSAFY 출석 체크 알리미 (크롬 익스텐션)

SSAFY 교육생이 **입실(09:00 이전)·퇴실(18:00 이후)** 체크를 놓치지 않도록 도와주는 크롬 익스텐션입니다.

> 퇴실 버튼은 미리 눌러도 되지만, **18:00 이후에 누른 기록이 있어야** 정상 퇴실로 인정됩니다. 18시 이전 기록만 있으면 조퇴 처리될 수 있습니다.

## 기능

### 🟥 빨간 사각형 강조 (콘텐츠 스크립트)
`edu.ssafy.com` 접속 시 **입실/퇴실 버튼 위에 네모 박스를 겹쳐 그려서** 지금 눌러야 할 버튼을 알려줍니다. 박스는 버튼의 실제 화면 위치를 따라가며(스크롤해도 유지), 상황에 따라 색과 문구가 바뀝니다.

| 상황 | 동작 |
|---|---|
| 평일 09:00 이전 + 입실 안 함 | 입실 버튼에 **빨간 박스** + "입실 체크! 09:00 마감 (남은 시간)" 라벨 |
| 평일 09:00 이후 + 입실 안 함 | 입실 버튼에 **빨간 박스** + "입실 체크 안 됨! 지금 체크" 라벨 |
| **입실 버튼 클릭 후** | 입실 박스 사라짐 (시각과 무관, 오늘 하루 유지) |
| 평일 09:00~18:00 (입실 완료) | 퇴실 버튼에 **주황 점선 박스** + "퇴실은 18:00 이후에" 라벨 |
| 평일 18:00 이후 + 유효한 퇴실 기록 없음 | 퇴실 버튼에 **빨간 박스** + "지금 퇴실하세요!" 라벨 + 상단 경고 배너 |
| **18:00 이후에 퇴실 버튼 클릭 후** | 퇴실 박스 사라짐 |

> 빨간 박스는 깜빡이며 강하게, 주황 점선 박스는 "아직 누르지 말고 기다리라"는 안내용입니다. 박스는 클릭을 막지 않으므로 버튼은 그대로 누를 수 있습니다.

### 🛡️ 클릭 기록 기반 강조 (조퇴/지각 방지)
버튼을 누른 기록을 하루 단위로 저장(`localStorage`)해서, **실제로 눌렀는지**에 따라 박스를 켜고 끕니다.

- **입실**: 입실 버튼을 누르면 그 순간 입실 박스가 사라집니다. (09:00 이전이어도, 페이지 새로고침해도 오늘은 다시 뜨지 않음)
- **퇴실**: 18:00 이후에 누른 기록이 생길 때까지 계속 강조합니다. 18시 전에 미리 눌러둔 기록만으로는 꺼지지 않으며, 18시 이전 클릭 시 "18시 이후에 한 번 더 눌러야 인정된다"는 안내 토스트를 띄웁니다.

### 🔔 크롬 알림 (백그라운드)
SSAFY 페이지를 열어두지 않아도 평일에 알림이 옵니다. 알림을 클릭하면 SSAFY 출석 페이지가 열립니다.

- **08:50** — "09:00 전에 입실 체크하세요"
- **18:00** — "퇴실 버튼을 누르세요"

> 크롬이 실행 중이어야 알림이 옵니다. 주말(토·일)에는 강조/알림이 동작하지 않습니다.

### 🔄 업데이트 공지 전달 (background.js)
관리자가 **GitHub Release를 발행**하면, 그때 작성한 **릴리즈 노트(제목·내용)가 사용자에게 크롬 알림으로 그대로 전달**됩니다. 버전 숫자를 비교하는 게 아니라, "새 Release가 올라왔는지"로 판단합니다.

- 확인 대상: `https://api.github.com/repos/park-rudxo/ssafy-check/releases/latest`
- 확인 주기: 6시간마다(+브라우저 시작 시). 새 Release가 있으면 알림 1회
- 알림 클릭 또는 팝업의 **`업데이트 확인`** 버튼 → 릴리스 페이지(전체 노트 + 다운로드)로 이동
- 크롬 확장은 압축해제(개발자 모드) 설치라 **자동 재설치는 불가**하므로, 공지를 받으면 최신 ZIP을 받아 새로고침(🔄)하는 방식입니다.
- 설치 직후에는 "현재 최신 Release"를 이미 본 것으로 기준을 잡아, 방금 설치한 사용자에게 곧바로 알림이 뜨지 않습니다.

#### 🚀 배포(공지)하는 법 (관리자)
코드를 main에 올리는 것만으로는 사용자에게 공지가 가지 않습니다. **GitHub Release 발행이 곧 배포 버튼**입니다.

1. (코드 변경이 있으면) `manifest.json`의 `"version"`을 올리고 커밋/머지
2. GitHub → **Releases → Draft a new release**
3. 태그 지정(예: `v1.1.1`), **제목과 릴리즈 노트에 사용자에게 보여줄 공지 내용**을 작성
4. **Publish release** 클릭 → 이 제목/노트가 사용자 알림으로 전달됨
   - (Draft로 두면 전달되지 않음. 준비되면 Publish)

### 🛠️ 개발자 모드 (팝업 하단)
실제 시간·출석 상태와 무관하게 강조 효과를 테스트할 수 있습니다. 툴바 아이콘 → **개발자 모드** 섹션을 펼쳐서 사용합니다.

- **개발자 모드 켜기**: 아래 오버라이드 설정을 활성화합니다.
- **가상 시각**: 지정한 시각인 것처럼 강조 로직을 동작시킵니다 (예: 08:30, 18:30).
- **입실 상태**: 자동 감지 / 입실 전 / 입실 완료 중 선택.
- **주말에도 평일처럼 동작**: 주말에 테스트할 때 사용.
- **원클릭 미리보기**:
  - `🟥 입실 강조` — 08:30·입실 전 시나리오로 입실 버튼 강조를 즉시 확인
  - `🟥 퇴실 강조` — 18:30·입실 완료 시나리오로 퇴실 버튼 강조를 즉시 확인
  - `⏹ 미리보기 끄기` — 실제 시간·상태 기준으로 복귀

> 강조는 **열려 있는 `edu.ssafy.com` 탭**에 나타납니다. 설정은 `chrome.storage.local`에 저장되어 팝업과 페이지가 실시간으로 연동됩니다. 실제 사용 시에는 개발자 모드를 꺼두세요.

## 설치 방법

1. 이 저장소를 다운로드(또는 `git clone`) 합니다.
2. 크롬 주소창에 `chrome://extensions` 입력 후 이동합니다.
3. 우측 상단 **개발자 모드**를 켭니다.
4. **압축해제된 확장 프로그램을 로드합니다** 버튼을 누르고, 이 저장소 폴더를 선택합니다.
5. `edu.ssafy.com`에 접속하면 자동으로 동작합니다.

## 파일 구성

```
manifest.json   # 익스텐션 설정 (Manifest V3)
content.js      # 출석 상태 감지 + 빨간 사각형 강조 + 조퇴 방지 가드
content.css     # 빨간 사각형/배너 스타일 (깜빡임 애니메이션)
background.js   # 08:50 / 18:00 크롬 알림 스케줄러 + GitHub Release 업데이트 확인
popup.html/js   # 현재 상태 요약 + 버전/업데이트 확인 + 개발자 모드(미리보기)
icons/          # 익스텐션 아이콘
```

## 동작 원리

- 페이지의 텍스트("입실", "퇴실하기", "정상 출석")를 기반으로 버튼과 출석 상태를 찾기 때문에 SSAFY 페이지의 세부 구조가 조금 바뀌어도 동작합니다.
- 15초마다 + DOM 변경 시마다 상태를 다시 확인합니다.

## 주의사항

- 이 익스텐션은 **자동으로 버튼을 눌러주지 않습니다.** 출석 체크는 본인이 직접 해야 하며, 익스텐션은 놓치지 않도록 시각적으로 알려주는 보조 도구입니다.
- 공휴일은 인식하지 못합니다 (평일이면 항상 동작).
